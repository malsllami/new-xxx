# دليل النشر - نظام تسجيل الدخول والتسجيل

## البنية
```
backend/
  Code.gs      → المنطق الرئيسي (تسجيل، دخول، لوحة المدير)
  Setup.gs      → يُشغّل مرة واحدة فقط لإنشاء الشيتات
frontend/
  index.html    → صفحة تسجيل الدخول (الصفحة الرئيسية)
  register.html → صفحة التسجيل
  admin.html    → دخول ولوحة تحكم المدير
  assets/
    style.css
    config.js   → يحتوي على رابط API (تحتاج تعدّله)
```

## الخطوة ١: إنشاء الباك إند (Google Apps Script)

1. اذهب إلى [sheet.new](https://sheet.new) لإنشاء Google Sheet جديد
2. من القائمة: **الإضافات (Extensions) → Apps Script**
3. سيفتح محرر Apps Script، احذف أي كود موجود في `Code.gs` الافتراضي
4. أنشئ ملفين بنفس الأسماء بالضبط: `Code.gs` و `Setup.gs`
5. الصق محتوى كل ملف من مجلد `backend/` المرفق

## الخطوة ٢: تشغيل الإعداد الأولي

1. من القائمة المنسدلة فوق محرر الكود، اختر دالة `setup`
2. اضغط **▶ تشغيل (Run)**
3. أول مرة، Google ستطلب منك الموافقة على الأذونات → اقبل
4. اذهب إلى **التنفيذ (Execution log)** في الأسفل، وستجد سطر مثل:
   ```
   رمز المدير الخاص بك هو: 482917
   ```
   **احفظ هذا الرمز فوراً** - هذا هو رمز دخولك كمدير، ولن يظهر مرة أخرى تلقائياً
   (لكن يمكنك دائماً فتح شيت "الإعدادات" ومشاهدته أو تغييره من خلية B2)

## الخطوة ٣: نشر الباك إند كـ Web App

1. من محرر Apps Script: **Deploy → New deployment**
2. اضغط على أيقونة الترس بجانب "Select type" واختر **Web app**
3. الإعدادات:
   - Execute as: **Me**
   - Who has access: **Anyone**
4. اضغط **Deploy**
5. سيظهر لك رابط ينتهي بـ `/exec` — **انسخه**

> ⚠️ كل مرة تعدّل على الكود وتريد نشر التحديث، تحتاج **Deploy → Manage deployments → ✏️ تعديل → New version → Deploy**
> النشر العادي (Save) لا يكفي لتحديث الرابط المنشور.

## الخطوة ٤: ربط الفرونت إند بالباك إند

✅ تم ربط رابط API الخاص بك مسبقاً في `frontend/assets/config.js` (الرابط الذي زوّدتني به).
إذا أعدت النشر (Deploy → New version) لاحقاً والرابط تغيّر، حدّث القيمة هناك.

## الخطوة ٤.٥: ضبط دومين GitHub Pages (مطلوب لتشغيل البصمة الحيوية فقط)

البصمة الحيوية (WebAuthn) تتطلب أن يعرف الخادم دومين موقعك الفعلي للتحقق من مصدر الطلب:

1. بعد نشر الموقع على GitHub Pages (الخطوة ٥) واستلام الرابط، مثلاً:
   `https://malsllami.github.io/myapp`
2. افتح محرر Apps Script → ملف `Setup.gs` → دالة `setGithubDomain()`
3. غيّر القيمة `'username.github.io'` إلى دومينك الحقيقي **بدون https:// وبدون أي مسار بعده** (فقط `malsllami.github.io`)
4. شغّل الدالة مرة واحدة

> إذا تخطّيت هذه الخطوة، تسجيل الدخول العادي ولوحة المدير يعملان بشكل طبيعي، لكن البصمة الحيوية لن تعمل وستظهر رسالة "لم يتم ضبط دومين الموقع".

## الخطوة ٥: النشر على GitHub Pages

1. أنشئ مستودع (repo) جديد على GitHub
2. ارفع كل محتويات مجلد `frontend/` (وليس مجلد frontend نفسه، بل محتوياته) إلى جذر المستودع
3. من إعدادات المستودع: **Settings → Pages**
4. Source: اختر **Deploy from a branch** → Branch: `main` → Folder: `/ (root)`
5. احفظ، وانتظر دقيقة، سيعطيك رابط مثل:
   ```
   https://username.github.io/repo-name/
   ```

## ⚠️ تحذير مهم: تغيير صيغة تخزين الجوال

تم تحديث النظام لتخزين الجوال بصيغة "كود الدولة + الرقم بدون صفر" (مثلاً `966512345678` بدل `0512345678`).
**أي مستخدم سجّلته قبل هذا التحديث لن يتطابق مع الصيغة الجديدة.** افتح شيت "المستخدمون" وصحّح العمود "الجوال" يدوياً لأي صف قديم (حذف الصفر من البداية، وإضافة 966 قبله)، أو اطلب من المستخدم التسجيل من جديد.

## ملاحظات حول البصمة الحيوية (WebAuthn)

- تعمل فقط عبر **HTTPS** (GitHub Pages يوفر هذا تلقائياً) - لن تعمل أبداً لو فتحت الملفات محلياً بصيغة `file://`
- "تفعيل البصمة لهذا الجهاز" يجب أن يتم **مرة لكل جهاز** بعد تسجيل دخول عادي بالجوال أولاً
- إذا غيّر المستخدم جهازه، يحتاج تفعيل البصمة من جديد على الجهاز الجديد (تسجيل الدخول العادي بالجوال يبقى يعمل كبديل دائم)
- اختبر تدفق التسجيل والدخول بالكامل على جهاز iPhone حقيقي عبر Safari قبل اعتماد هذه الميزة، لأن هذا تنفيذ مبني من الصفر (CBOR + ECDSA) وليس مكتبة جاهزة - حافات الحالات النادرة (مثل بعض صيغ توقيع DER) قد تحتاج تعديلاً بعد أول اختبار فعلي

## الاختبار

1. افتح الرابط → اضغط "سجّل الآن" → سجّل ببيانات تجريبية
2. افتح `admin.html` (أو اضغط "دخول المدير" من الصفحة الرئيسية) → أدخل رمز المدير
3. ستشاهد الطلب في تبويب "قيد المراجعة" → اضغط "موافقة"
4. رجوع للصفحة الرئيسية → سجّل دخول بنفس رقم الجوال → يجب أن يعمل الآن

## ملاحظات أمنية مهمة

- **هذا النظام مناسب لمجموعة مغلقة (عائلة/مجتمع صغير)** وليس لموقع عام مفتوح، لأن تسجيل الدخول يعتمد على رقم الجوال فقط بدون كلمة مرور أو رمز تحقق (OTP)
- رمز المدير يُرسل مع كل طلب من لوحة التحكم - هذا مقبول لمشروع بهذا الحجم، لكن لا تشارك الرابط `/exec` أو الرمز علناً
- إذا أردت رفع مستوى الأمان لاحقاً: أضف تحقق OTP عبر SMS عند تسجيل الدخول، أو استخدم نظام WebAuthn (بصمة/Face ID) الذي بنيته في مشاريعك السابقة - نفس الكود قابل لإعادة الاستخدام هنا
